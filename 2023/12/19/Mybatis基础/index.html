Mybatis基础

###一、基本概念

  • 1、层的概念

    Mybatis是一个用来操作数据库的持久层（即Dao层）框架。这里重新回顾一下关于Dao层，Service层，Controller层的关系。

    • Dao层：用于完成持久化工作，即实现将即时数据存储到文件或是数据库中的操作的代码块。
    • Service层：用于封装业务需求的逻辑，调用Dao层将数据固化后返回结果给Controller层。
    • Controller层：接入用户端的层，通过post/get接受和反馈用户的请求，过程中传入数据并调用Service层去完成工作。

###二、Mybatis使用流程

  • 1、Mybatis的获取、安装（后面很多东西都在这两个地方找）

    • Github：在github里搜mybatis就有各种相关文章，下载framework3.0就行。
    • Maven repository：MVN仓库里找mybatis，复制xml配置。

  • 2、使用思路（框架使用都是这个）：搭环境 → 导框架的包 → 写代码 → 测试

    • 2.1—前置准备：确定要操作的数据库，建好表和字段。

    • 2.2—创建项目：创建Maven项目，IDEA2021版本以上的，创建时是Maven Archetype（Maven骨架，quickstart是普通maven的java项目，webapp是web项目），创建选项注意组ID和工程ID一样就行。之后删除src文件夹，让工程作为父工程。

    • 2.3—初始化：导入插件和依赖，检查JDK和Maven版本，Maven仓库配置。

      1. 首先检查JDK和Maven仓库:在IDEA左上角文件 → 设置/项目结构检查Maven版本/JDK版本。

      2. 检查插件/依赖:打开pom.xml文件，翻到下面有和标签，分别对应依赖和插件两个模块的配置。

      3. 导入依赖: 前往Maven Repository: Search/Browse/Explore (mvnrepository.com)Maven仓库官网搜索需要的依赖驱动，比如mysql，Junit，mybatis驱动包，复制Maven框内的内容到组内，刷新即可。

        要检查是否下载好依赖和插件，在IDEA右边Maven侧边栏点开查看有没有就行。

    • 2.4—创建模块：删除src文件夹的意义就是在于让你这个项目下创建的模块能作为子工程，直接使用父工程的pom.xml配置。

    • 2.5—配置核心文件和工具类：创建模块后第一件事是配置好核心文件和mybatis要用的工具类。这些配置最好还是参照官方指导手册mybatis – MyBatis 3 | 入门

            1. 本次教程的核心在于如何用Mybatis实现Java项目和数据库之间的交互，XML映射的这一部份是只关于“交互”上的操作。
            2. 在子工程的配置文件中已经通过URL确定了该工程模块连接的是数据库的哪张表，这时Mybatis已经是识别出表里的字段内容了，要操作这些字段我们就要借助实体类
            3. 创建一个实体类，里面的成员变量属性要和表中字段完全一致，为的是Mybatis能将表中的字段数据匹配到这个类中的成员变量数据中去。然后就能通过操作这个类中的成员变量来控制表中的数据变化，所以这个实体类要写好JavaBean封装。
            4. 创建一个操作实体类的接口，用“同名Mapper”命名，Mapper内写目的函数。在JDBC中该Mapper下应存放各种SQL语句函数，比如要写一个查询语句就要建一个查询语句的函数，在函数里面写查询的sql语句。而在Mybatis中省略了后面的那一步，只要建好函数就行，函数内的sql语句会放到XML文件中映射执行。
            5. 创建Mapper的对应XML映射文件。xml文件内头部约束自行查找，里面就和[Mybatis入门][#Mybatis入门.md]写的一样。mapper内写的sql语句就和在navicat写的一样作用了，数据的变化是一样的，只是操作结果通过java项目内的形式返回。
            6. 故整体流程是：模块xml配置识别数据库 → pojo实体类匹配数据表字段 → Mapper接口内方法借pojo类预留字段对应类型的存储空间G → Mapper映射xml文件绑定Mapper接口，预编译具体sql语句，返回结果存放到空间G → 测试程序获取工具类的sqlsession对象 → sqlsession对象用Mapper接口的class属性获取Mapper接口对象 → 使用Mapper接口对象对应方法获取空间G所存放的结果 → 输出结果。

    • 2.6—代码测试：以上，写完代码后就用Junit来测试代码是否能正常运行 → 在test包下的绿色java包内，和mapper同级同名一路建包建到MapperTest，Test类内先通过工具类方法获取sqlsession对象，然后session对象用getMapper方法读Mapper的class属性来获取对应的Mapper对象 → 调用Mapper对象的方法，打印。
